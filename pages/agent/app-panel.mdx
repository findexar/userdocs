# App

The **App** tab shows the **embedded interface** for whichever **creator app** is currently in focus—often the same app that produced a **tool** or **widget** from chat.

## What you see

- A **live panel** loaded from the app (HTML UI hosted through Agentsyx). The exact layout depends on the **creator’s** app.
- Sometimes a **list of widgets or views** if the app exposes more than one panel—you can switch between them from controls inside the tab.
- Optional actions such as **reload** / **reset view** (wording varies by version) if the creator enabled them.

## Subscription and trials

If you **don’t** have the right **plan** or **entitlement**, the **App** tab may show a **subscription required** state, **pick a plan**, or **start a free trial** when the creator configured a **zero-price** trial plan.

Details and other places plans appear: **[Subscriptions](/subscriptions)**.

## Relationship to **Agent (chat)**

When a chat **tool** returns a **widget**, the UI often **moves you to the App tab** automatically so you can use the visual UI. After you finish, go back to **[Agent (chat)](/agent/chat)** to continue the conversation.

## API key and support (when shown)

Some apps surface **your Agentsyx API key** (masked) or links to **documentation**, **integrations**, or **report an issue**. Those sections are provided by the **creator** for that app.

## Cache / reload

If a panel looks **stale** after the creator shipped an update, use any **reset cache** or **reload** control shown in the tab, or reload the page.
